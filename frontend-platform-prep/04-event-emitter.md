# EventEmitter

Цель: уметь реализовать простой pub/sub и не ошибаться с контрактами `on/off/emit`.

## Что нужно уметь

- Хранить подписчиков по имени события (обычно `Map<string, Set<fn>>` или `Map<string, fn[]>`).
- Возвращать `this` из `on/off/emit` для chaining (если это часть контракта).
- Корректно:
  - удалять конкретный callback;
  - не падать, если отписка/эмит по несуществующему событию;
  - передавать аргументы в callback.

## Частые вопросы

- `emit` должен возвращать boolean или `this`? (зависит от API — важно проговорить контракт)
- Как реализовать `once`?
- Что делать, если callback отписывает сам себя во время `emit`?
